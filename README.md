# 🎓 Мини-CRM для образовательного агентства (MVP)

Проект разработан в рамках учебной дисциплины «Основы проектной деятельности» (ОПД). 
Представляет собой локальную CRM-систему для учета студентов и безопасного хранения сканов их документов. Приложение спроектировано как MVP (Minimum Viable Product) для использования одним сотрудником на локальном ПК.

## 🌟 Ключевой функционал
* **Картотека студентов:** Вывод полного списка зарегистрированных студентов.
* **Добавление записей:** Форма для ввода данных (ФИО, паспорт, программа обучения, адрес).
* **Менеджмент документов:** Безопасная загрузка и локальное сохранение сканов паспортов (форматы `.pdf`, `.jpg`, `.png`).
* **Мгновенный поиск:** Фильтрация таблицы "на лету" с помощью клиентского JavaScript без дополнительных запросов к серверу (Zero-latency UX).

## 🛠 Технологический стек и обоснование выбора
* **Backend:** `Python 3` + `Flask`. Легковесный и гибкий фреймворк, идеально подходящий для быстрого развертывания MVP без избыточного бойлерплейта.
* **Database:** `SQLite`. Выбрана для обеспечения максимальной автономности. Не требует установки, настройки и поддержки отдельного сервера БД. База хранится локально в одном файле, что идеально для однопользовательского оффлайн-приложения.
* **Frontend:** `HTML5`, `Bootstrap 5` (для создания чистого и адаптивного UI), `Vanilla JavaScript` (для мгновенной фильтрации DOM-элементов).
* **Security:** Использование утилиты `secure_filename` из библиотеки Werkzeug для защиты от уязвимостей типа Path Traversal при загрузке файлов.
